1) PROJECT OVERVIEW
- Project type: Node.js + TypeScript + Express + Prisma + MySQL
- Purpose: Backend auth microservice integrating AWS Cognito through custom APIs.
- API base path: /api/v1
- Health endpoint: GET /health

2) WHAT HAS BEEN IMPLEMENTED
The following Cognito-backed flows have been added:
- User sign up
- Email verification (confirm sign up)
- Registration approval/rejection workflow
- Approved-user provisioning into internal user table
- Login (username + password)
- MFA challenge response
- Forgot password
- Confirm forgot password

All flows are exposed via backend APIs so frontend interacts only with this project.

3) NEW AUTH API ENDPOINTS
Base: /api/v1/auth

- POST /signup
  Input: username, first_name, last_name, email, phone, password
  Action: Creates user in Cognito, sends verification code.

- POST /signup/confirm
  Input: username, confirmation_code
  Action: Confirms user in Cognito, reads Cognito attributes, creates/updates register_user with PENDING_APPROVAL.

- POST /registrations/:id/review
  Input: action (APPROVE/REJECT), review_note, approved_by
  For APPROVE also requires: role_id, site_id, corporation_id
  Action:
  - REJECT -> mark register_user as REJECTED
  - APPROVE -> mark register_user as APPROVED and create app_user record

- POST /login/initiate
  Input: username, password
  Action:
  - Validates user exists in app_user and is ACTIVE
  - Calls Cognito USER_PASSWORD_AUTH
  - Returns either tokens (if no challenge) or challenge_name + session (if MFA required)

- POST /login/respond
  Input: username, session, challenge_name, challenge_code
  Action: Responds to Cognito challenge and returns auth tokens.
  Supported challenges in code:
  - SMS_MFA
  - SOFTWARE_TOKEN_MFA
  - EMAIL_OTP

- POST /forgot-password
  Input: username
  Action: Triggers Cognito forgot-password code delivery.

- POST /forgot-password/confirm
  Input: username, confirmation_code, new_password
  Action: Confirms password reset in Cognito.
